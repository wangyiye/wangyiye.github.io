JSX

javascript 和 xml(html) 虚拟的DOM

  • 只能有一个根元素节点 不要用HTML和BODY
  • 基于{}或者JS表达式
    • 可以识别null/undefined
    • 不能直接使用引用数据类型,除了数组，数组会转化为字符串
    • 可以使用表达式返回有效的数据值或者返回JSX元素
  • JSX元素设置样式
    • 设置样式类 className
      • className =’box’ 值是字符串
      • style={color} 值是对象

      这里可以找到传进来的所有子元素 })

    React脚手架中设置了@babel/plugin-proposal-class-properties 可以直接在类方法中 最外层写属性，会添加到实例上

    • 必须要有render函数，里面返回的是要渲染的虚拟dom元素（视图）
    • REACT-DOM.RENDER 渲染的时候，如果发现虚拟DOM中TYPE是一个类组件，会创建这个类的一个实例，并且把执行的参数传给这个类（props） new Class(props)
      1. 先执行constructor,并未挂载到实例（此时无法通过this。props获取到属性，super（props，context,ref）把父类当普通函数执行this指向当前实例，后即可通过this.props/context/refs）
        • 引入 prop-types 插件 
    2. React 把props/context/refs挂载到实例上
3. render方法执行返回视图中jsx虚拟dom元素

生命周期函数

  1. 获取属性 static defaultProps/ static propsTypes
  2. constructor 构造函数执行获取到state状态信息
  3. componentWillMount 组件即将挂载（异步获取数据改变状态信息）
  4. render 渲染
  5. componentDidMount 可以获取到dom元素，组件已经挂载到页面中
  6. componentWillReceiveProps(nextProps)
  7. shouleComponentUpdate状态改变触发返回true继续false停止(两个参数 nextProps,nextState)修改后的属性和状态(当状态是通过forceUpdate强制更新不会走这一步)
    这里可以做性能优化，react中setState的方法执行，无论状态值是否改变都会使组件重新渲染，造成无用的消耗，可以基于此优化该过程，React.PureComponent源码中就是对该方法进行了重新的配置，将nextState与之前的state信息进行浅比较，相同就返回false让渲染不进行，反之继续。PureComponet组件中几点要注意
    1. 如果自己设置了shouldComponentUpdate,以自己设置为主。
    2. 是浅比较，第一层，如果是引用数据类型，即使更改了里面的某个键值对，也不会判定为不同，因为比较的是引用地址，如果地址不发生变化，就会判定为相同。
    3. PureComponent组件对forceUpdate无效，forceUpdate不走shouldComponentUpdate这一钩子函数，直接跳到ComponentWillUpdate
    4. 如果父组件是PureComponent子组件也会具有统一效果，因为父组件不渲染，子组件也不会渲染。
    5. 只适用于属性和状态不频繁更新的组件，用来做优化，相反如果频繁被变化的，会对阻碍性能。以为每次都会进行比较消耗性能。

  1. 基于props属性传递单向父组件=》子组件

  2. 子组件操作修改父组件状态信息

    • callback（jsonp/jsbridge)父组件把自身的方法通过属性传递给子组件，子组件执行传递进来的方法，通过方法把子组件信息传递给父组件。子组件中执行onClick ={callback.bind(this.xxx)}/ onClick = {ev =>{callback(xxx)} // bind执行的话最后一个参数默认把ev传进去
    • 写eventEmit方法或者引用eventEmit发布订阅插件
      • $on订阅方法放到拟事件池中
      • $emit通知执行
    1. 向事件池中添加方法执行eventEmit中的$on方法在一般在componentDidMount中将自定义事件添加到pond中
    2. 在要执行的方法下 执行 $emit(eventName,xxx)方法
