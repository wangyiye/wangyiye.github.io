创建的项目下的文件夹详情

  • public
    _ index.html(SPA各组件最后渲染完后的结果都放入此页面的#root盒子中) ![Alt

    _ xxx.html(MPA/SPA 存放编译页面的模板)
    _公共资源（直接基于link的方式导入到模板当中不是基于webpack合并到一起，项目中某些模块不支持CommonJS规范/es6module不能import导入，只能在此引入使用）

  • src （整个项目的源码存放目录）

    • index.js （当前项目入口，webpack从这个文件开始导入打包（MPA中创建多个入口文件即可）
    • api
    • store REDUX公共状态管理
    • assets 存储公共i资源
    • route 路由管理
    • utils （公共JS模块）
    • components 公共组件

  • package.json （依赖清单）

    • "react": "^16.10.2", react核心，提供 状态/属性/组件/生命周期等 "react-dom": "^16.10.2", JSX语法渲染成为真实DOM显示在浏览器中 "react-scripts": "3.2.0" webpack配置项，隐藏在node_module中，脚本执行通知webpack打包编译 
    • > 执行 yarn eject 前提交历史版本不可逆操作
      • eject
        • babel-preset-react-app 解析JS语法
    • 修改默认域名端口

  • config

    • webpack.config.js 修改合并压缩依赖的插件和加载器配置
      • less重新配置 下载 less less-loader

JSX

javascript 和 xml(html) 虚拟的DOM

  • 只能有一个根元素节点 不要用HTML和BODY
  • 基于{}或者JS表达式
    • 可以识别null/undefined
    • 不能直接使用引用数据类型,除了数组，数组会转化为字符串
    • 可以使用表达式返回有效的数据值或者返回JSX元素
  • JSX元素设置样式
    • 设置样式类 className
      • className =’box’ 值是字符串
      • style={color} 值是对象

组件(VScode中JS命名为JSX方便识别，内部文件已 .jsx处理· )

  1. 函数式组件 （函数执行，返回值，调用组件时候把属性（只读属性不可修改）传递进去）

    • 函数返回一个JSX元素

    • 静态组件一旦组件被调用渲染完成，当前组件中的信息不可变，没有状态信息，生命周期函数等

      • 给组件传递进来的属性屎只读属性，不可修改以及赋值默认值，可新定义变量接受
      • 不可以基于prop-types给属性设置默认规则

      组件双闭合会有一个children属性，在函数中可以直接props.children（1个子元素对象类型，2个是个数组）调用，可以通过React.Children.map(props,(item,index) =>{
      这里可以找到传进来的所有子元素 })

  2. React.Component类创建(类组件，受控组件）

    React脚手架中设置了@babel/plugin-proposal-class-properties 可以直接在类方法中 最外层写属性，会添加到实例上

    • 必须要有render函数，里面返回的是要渲染的虚拟dom元素（视图）
    • REACT-DOM.RENDER 渲染的时候，如果发现虚拟DOM中TYPE是一个类组件，会创建这个类的一个实例，并且把执行的参数传给这个类（props） new Class(props)
      1. 先执行constructor,并未挂载到实例（此时无法通过this。props获取到属性，super（props，context,ref）把父类当普通函数执行this指向当前实例，后即可通过this.props/context/refs）
        • 引入 prop-types 插件 
    2. React 把props/context/refs挂载到实例上
3. render方法执行返回视图中jsx虚拟dom元素

  3. 非受控组件(不受状态管控，直接操作DOM）

    + 组件行内 设置 ref = 'name' 内部直接调用this.refs.name //  ref = {  element =>{ this.name =element}}  直接this.name调用

生命周期函数

  1. 获取属性 static defaultProps/ static propsTypes
  2. constructor 构造函数执行获取到state状态信息
  3. componentWillMount 组件即将挂载（异步获取数据改变状态信息）
  4. render 渲染
  5. componentDidMount 可以获取到dom元素，组件已经挂载到页面中
  6. componentWillReceiveProps(nextProps)
  7. 状态改变触发shouleComponentUpdate返回true继续false停止(两个参数 nextProps,nextState)修改后的属性和状态(当状态是通过forceUpdate强制更新不会走这一步)
  8. componentWillUpdate
  9. render
  10. conponentDidUpdate
  11. componentWillUnmount(卸载组件）

复合组件之间的信息传递

  1. 基于props属性传递单向父组件=》子组件

  2. 子组件操作修改父组件状态信息

    • callback（jsonp/jsbridge)父组件把自身的方法通过属性传递给子组件，子组件执行传递进来的方法，通过方法把子组件信息传递给父组件。子组件中执行onClick ={callback.bind(this.xxx)}/ onClick = {ev =>{callback(xxx)} // bind执行的话最后一个参数默认把ev传进去
    • 写eventEmit方法或者引用eventEmit发布订阅插件
      • $on订阅方法放到拟事件池中
      • $emit通知执行
    1. 向事件池中添加方法执行eventEmit中的$on方法在一般在componentDidMount中将自定义事件添加到pond中
    2. 在要执行的方法下 执行 $emit(eventName,xxx)方法

  3. 执行上下文信息传递（祖先与后代组件之间传递）后代需要的信息全部放到上下文中，后代就可以调取使用

    • 同一祖先的上下文中

    • 后代组件直接调用
